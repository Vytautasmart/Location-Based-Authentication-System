## Build & Development Commands

```bash
# Install all dependencies (backend + frontend)
npm install && cd frontend && npm install && cd ..

# Development server (auto-reload with nodemon, builds frontend first)
npm run devstart

# Production start (builds frontend, then runs ./bin/www)
npm start

# Frontend only (Vite dev server on port 5173)
cd frontend && npm run dev

# Build frontend
npm run build-frontend

# Run tests (Jest + Supertest, --runInBand)
npm run test

# Lint frontend
cd frontend && npm run lint
```

## Architecture Overview

Location-Based Authentication System (LBAS): a full-stack app that grants/denies access based on user identity + GPS location + IP-spoofing checks.

### Tech Stack
- **Backend:** Express 5, Passport.js (Local + JWT), PostgreSQL (`pg`), bcryptjs, helmet, express-rate-limit, express-validator
- **Frontend:** React 19, Vite, React Router, Leaflet
- **Servers:** dual HTTP (3000) + HTTPS (3443) locally via [bin/www](bin/www); single HTTP on `process.env.PORT` when deployed behind a managed proxy (Render/Railway — detected via env vars)

### Core Flow (`POST /api/auth/access`)
1. Client submits `{ username, password, latitude, longitude }`
2. `passport-local` validates credentials (with account lockout + timing-attack-resistant dummy hash)
3. For non-admin users, [locationService.isLocationSpoofed](services/locationService.js) calls **ip-api.com** + **ipinfo.io** in parallel and compares IP-derived country to OSM/Nominatim reverse-geocoded country (falls back to 500 km distance threshold). Detects VPN/proxy/hosting.
4. [locationService.verifyLocation](services/locationService.js) checks the GPS point against the user's assigned zones (or all zones if none assigned), preferring 3 m microgrid (`w3w`) matches over circular radius zones
5. [authorizationService.grantAccess](services/authorizationService.js) makes the final decision — **admin role bypasses location checks**
6. On success: issue HS256 JWT (15 min) and rotate a 64-byte refresh token (bcrypt-hashed in DB, HTTPOnly+Secure+SameSite=strict cookie, 7-day expiry). Every attempt is written to `auth_logs`.

### Key Directories
- [routes/](routes/) — `auth.js` (login/access/refresh/logout), `users.js` (register, /me), `zones.js` (CRUD + w3w grid + user-zone assignments)
- [services/](services/) — `locationService.js`, `authorizationService.js`, `w3wService.js` (self-computed 3 m microgrid, no external API)
- [middleware/](middleware/) — `passport.js` (Local + JWT strategies, lockout), `rbac.js` (`checkRole`), `validation.js` (express-validator rules incl. dynamic w3w/circular zone validator)
- [db/](db/) — `postgre.js` pool + 8 migrations (V0–V7)
- [frontend/src/](frontend/src/) — `pages/` (Landing, Login, Register, Dashboard, Demo) + `components/` (Map, Navbar)
- [__tests__/](__tests__/) — `auth.test.js`, `zones.test.js`
- [bin/www](bin/www) — server bootstrap with proxy detection
- [ssl/](ssl/) — self-signed certs (regenerated by [generate-cert.js](generate-cert.js))

### Database Tables (migrations V0–V7)
- `users` — credentials + role (`admin` / `user`)
- `authorized_zones` — `type` is `circular` (lat/lng/radius) or `w3w` (center + grid squares)
- `w3w_zone_squares` — per-zone 3 m grid cells keyed by `words` (`row.col`)
- `user_zones` — many-to-many user ↔ zone assignments (V6)
- `auth_logs` — full audit trail (lat/lng, IP, IP geo, spoofing flags, latency)
- `refresh_tokens` — bcrypt-hashed token + expiry, rotated on every refresh
- `login_attempts` — drives 15-min / 5-attempt account lockout (V5)

### API Endpoints
| Endpoint | Auth | Description |
|----------|------|-------------|
| POST /api/auth/login | None (rate-limited) | Credential-only login → JWT |
| POST /api/auth/access | None (rate-limited) | Login + GPS + IP-spoof check → JWT + refresh cookie |
| POST /api/auth/refresh | Refresh cookie | Rotate refresh token, issue new JWT |
| POST /api/auth/logout | Refresh cookie | Invalidate refresh token |
| POST /api/users | None (rate-limited 10/hr) | Register user |
| GET /api/users/me | JWT | Current user profile |
| GET /api/zones | None | List zones (with w3w squares) |
| POST/PUT/DELETE /api/zones | JWT + admin | Manage zones (circular or w3w) |
| GET /api/zones/w3w/grid | JWT + admin | GeoJSON grid for a bbox |
| GET /api/zones/w3w/convert | JWT + admin | lat/lng → grid cell ID |
| GET/POST/DELETE /api/zones/:id/users | JWT + admin | Assign users to zones |

### Security Middleware (in [app.js](app.js))
- helmet w/ custom CSP (allows OSM tiles + unpkg leaflet assets), HSTS 1-year + subdomains, strict-origin-when-cross-origin referrer
- CORS — origins from `CORS_ORIGIN` env (comma-separated) or `https://localhost:3443,https://localhost:5173`; credentials enabled
- `trust proxy: 1`
- `express-rate-limit` — auth routes 100/15min, registration 10/hr (per IP)
- JSON/urlencoded body limit 1 MB
- Production error handler hides 500 details

## Environment Variables

Required in `.env`:
```
DATABASE_URL=postgresql://user:password@localhost:5432/dbname
JWT_SECRET=your_secret_key
```

Optional:
```
IPINFO_TOKEN=...           # raises ipinfo.io rate limit
CORS_ORIGIN=https://a,https://b
PORT=3443                  # HTTPS port locally; HTTP port when behind proxy
HTTP_PORT=3000             # HTTP redirect port locally
RENDER=true                # auto-set by Render — switches bin/www to plain HTTP
```
